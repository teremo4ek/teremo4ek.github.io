Решил перенести свой блог на Hugo и заодно разобраться, как устроены современные статические генераторы. Спойлер: это оказалось проще, чем я думал.

## Почему Hugo

Статические сайты быстро загружаются, не требуют базы данных и легко деплоятся на GitHub Pages. Hugo выделяется среди аналогов скоростью сборки — даже большие сайты генерируются за секунды.

## С чего начать

1. Установить Hugo Extended (нужен для Tailwind CSS и PostCSS)
2. Выбрать тему или начать с шаблона — я взял Hugoplate
3. Настроить конфиг и наполнить контентом
4. Деплой через GitHub Actions — буквально пара строк
